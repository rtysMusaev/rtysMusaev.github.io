Инструкция по работе с git


Инициализация

git init - инициализация гита в репозитории

Статус

git status - проверить состояние гита

Отслеживание файлов гитом

git add полноеНазваниеФайла - добавить файл в отслеживаемые (необходимо вводить перед каждым коммитом)
git add . - добавить в отслеживаемые все файлы в каталоге

Коммиты

git commit -m "комментарий" - произвести коммит. Позволяет сохранить изменения в новой версии

Работа с прошлыми версиями проекта

git log - просмотреть список всех коммитов.
Флаг --oneline - один коммит выводится в одну строку.
Флаг --graph - коммиты выводятся в виде дерева.
Эти флаги можно комбинировать.
git checkout номерВерсии - вернуться к версии
git diff - просмотреть разницу между текщим и последней версиями

Ветки

Ветвление стало неотъемлемой частью командной разработки, потому что оно дает возможность работать над разными версиями исходного кода. Основной идеей ветвления является отклонение от основного кода и продолжение работы независимо от него. Также это удобно в тестировании отдельного функционала, потому что позволяет работать над новой частью кода, не беспокоясь о поломке чего-то в рабочей версии.
git branch - отобразить список веток (показывает также текущуюю ветку)
git branch названиеВетки - создать новую ветку
git branch -M названиеВетки - переименовать
git branch -d названиеВетки - удалить ветку
git branch -D названиеВетки - принудительно удалить ветку

Слияние веток

git merge название ветки - слить ветку с текущей

Работа с удаленными репозиториями

Удалённые репозитории представляют собой версии вашего проекта, сохранённые в интернете или ещё где-то в сети. У вас может быть несколько удалённых репозиториев, каждый из которых может быть доступен для чтения или для чтения-записи. Взаимодействие с другими пользователями предполагает управление удалёнными репозиториями, а также отправку и получение данных из них. Управление репозиториями включает в себя как умение добавлять новые, так и умение удалять устаревшие репозитории, а также умение управлять различными удалёнными ветками, объявлять их отслеживаемыми или нет и так далее.
git clone ссылка - скопировать проект из сети по ссылке
git remote add присваемоеИмя ссылка - добавить удаленный репозиторий с указанным именем, по указанной ссылке
git push присваемоеИмя названиеВетки - добавить изменения в удаленный репозиторий в указанную ветку
Если указанная ветка новая и ее нет в удаленном репозитории то стоит использовать флаг -u для создания ветки в удаленном репозитории.
git pull - загрузить все изменения из удаленного репозиториями
